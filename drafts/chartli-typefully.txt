Introducing chartli 📊

A CLI for terminal charts from numbers.

ascii, spark, bars, columns, heatmap, unicode, braille, svg.

$ npx chartli

I wanted zero-setup terminal charts: no browser, no Python env, no matplotlib. Pipe numbers in, get a chart out.

Built with Command Code.
---
Example:

$ npx chartli data.txt -t ascii -w 24 -h 8

The input format is intentionally dead simple: rows of space-separated numbers. Multiple columns = multiple series.
---
0.0 0.1 0.1 0.1
0.2 0.4 0.2 0.4
0.3 0.2 0.4 0.2

That is enough to generate multi-series terminal charts.
---
It composes nicely with pipes:

$ cat metrics.txt | chartli -t spark

S1 ▁▂▃▄▅▆
S2 ▁▄▂▇▅█
S3 ▁▂▄▃▆▅
S4 ▁▄▂▇▂▇
---
There are 8 chart types spanning a fun range of Unicode density:

- ascii: line charts with ○◇◆● markers
- spark: ▁▂▃▄▅▆▇█ sparklines, one row per series
- bars: horizontal bars with ░▒▓█ shading per series
- columns: vertical grouped bars
---
- heatmap: 2D grid with intensity mapping
- unicode: grouped bars with ▁▂▃▄▅▆▇█ sub-cell resolution
- braille: 2x4 dot matrix, highest density
- svg: vector output, circles or polylines
---
The braille renderer is my favorite.

Each braille character encodes a 2x4 dot grid, so a 16-wide chart gives you 32 pixels of horizontal resolution.

Free anti-aliasing from Unicode.
---
The bars renderer uses 4 shading levels (░▒▓█) to visually separate series without color.

This works on basically any terminal, any font, and still reads well in monochrome setups.
---
Heatmap maps values to a 5-step intensity scale across a row x column grid, so patterns in tabular data pop out immediately.

SVG mode has 2 render paths: circles or lines.

The output is valid XML, so you can pipe it straight to a file or another tool.
---
Zero config by default. Every dimension is overridable:

-w width
-h height
-m SVG mode

No config files.
No themes.
No dashboards.
---
Just:

$ npx chartli

Or install it globally:

$ npm i -g chartli
---
